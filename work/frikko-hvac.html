The Challenge: Simplifying a Complex Buying Journey

Frikko possessed an extensive product catalog with highly complex technical specifications that easily overwhelmed the average consumer. The main challenge was to reduce cognitive friction when users searched for the right air conditioning unit.

We needed to modernize the user interface (UI) and engineer a robust, self-manageable backend for their marketing team—without sacrificing crucial page load times (WPO).

Development Methodology & UX Strategy

1

1 UX & Information Architecture

We initiated the project with a competitive benchmark in the HVAC sector. We identified a systemic failure among competitors: the inability to present heavy technical data in a digestible format for B2C consumers. We wireframed a new catalog logic and implemented advanced filtering to drastically reduce friction in the purchasing journey.

2

2 UI Design & Scalable System

According to the Aesthetic-Usability Effect, users perceive aesthetically pleasing interfaces as easier to use. We built a scalable Design System in Figma, prototyping high-fidelity, modular components to ensure visual consistency and prepare a structured handoff for the development phase.

3

3 Advanced WordPress Engineering

  • Data Architecture (ACF Pro): Implemented custom fields to dynamically manage comparison tables, BTU metrics, and technical PDF downloads.
  • Modular Development: Created dynamic single-product templates using clean, semantic builders (Bricks) to avoid DOM bloat.
  • WPO & Speed: Aggressive image optimization (WebP), lazy loading, and caching to dominate Google PageSpeed Insights.
