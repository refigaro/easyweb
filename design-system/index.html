Container Component

Este sistema define un componente CSS reutilizable .container. Su comportamiento base es width: 95vw y desde min-width: 1920px aplica max-width: 1600px. Los elementos .section usan padding vertical consistente y son siempre el parent directo de un .container.

Regla Width Se aplica en Visual
Base 95vw 0px - 1919px
Cap min(95vw, 1600px) min-width: 1920px 
.container {
  width: min(100%, var(--container-width));
  margin-inline: auto;
}

@media (min-width: 1920px) {
  .container {
    max-width: var(--container-max-width);
  }
}

.container--content {
  width: min(100%, var(--content-max));
}

/* Uso */
.hero,
.features,
.cta {
  /* Cualquier sección reutilizable */
}

.hero .container,
.features .container,
.cta .container {
  /* El mismo wrapper de layout en todo el sitio */
}
Section Component

.section define el ritmo vertical del layout con --space-24 arriba y abajo. Regla del sistema: cada .section debe envolver directamente un .container. Variantes disponibles: .section--tight (usa --space-16) y .section--hero (usa --space-32).

Base: .container dentro de .section 
.section {
  padding-top: var(--space-24);
  padding-bottom: var(--space-24);
}

.section--tight {
  padding-top: var(--space-16);
  padding-bottom: var(--space-16);
}

.section--hero {
  padding-top: var(--space-32);
  padding-bottom: var(--space-32);
}

/* Estructura obligatoria */
<section class="section">
  <div class="container">
    ...contenido...
  </div>
</section>

/* Variantes */
<section class="section section--tight">...</section>
<section class="section section--hero">...</section>
Variante Padding vertical Uso recomendado Visual
.section--tight --space-16 Bloques densos o listados
.section (base) --space-24 Secciones estándar del sitio
.section--hero --space-32 Hero, CTA principal o separadores fuertes

Animar solo transform y opacity. Nunca transition-all. Pasa el cursor sobre las tarjetas para ver cada easing en acción.

Smooth
--ease-smooth

Ease Out
--ease-out
cubic-bezier(0, 0, 0.2, 1)

Tablas

El sistema incluye dos tipos de tablas según el caso de uso: tablas de especímenes (con <div> + CSS Grid) para documentación visual, y tablas de datos (con <table> semántica) para datos tabulares reales.

Tabla de especímenes

Cuándo usar: documentación de design tokens, especímenes tipográficos, o cualquier layout que requiera metadata apilada (label + specs + token) y control preciso del diseño. No son datos tabulares, son especímenes visuales.
